# 뉴스 섹션 개선 현황 (비개발자용)

작성일: 2026-02-22

## 1) 이번에 완료한 것

1. 뉴스 소스에서 `블루밍비트/코인속보` 의존을 제거했습니다.
2. 기존 DB에 남아있던 해당 소스 뉴스도 삭제했습니다.
3. 뉴스를 "많이"보다 "신선하고 중요한 것" 위주로 고르도록 바꿨습니다.
- 최근 6시간 중심
- 유사 제목 중복 억제
- 출처 신뢰도 + 자산 관련도 + 최신성 점수로 선별
4. AI 호출량 상한을 넣어 토큰 낭비를 줄였습니다.
- 상위 품질 뉴스만 AI 분석
- 나머지는 저비용 태깅
5. 뉴스 탭 3번째를 `내 자산 뉴스`로 변경했습니다.
- 내 보유 자산과 매칭되는 뉴스만 별도로 볼 수 있습니다.
6. 각 뉴스에 `내 자산 영향지수`를 표시합니다.
- 노출 비중 + 뉴스 방향(상승/하락 압력)을 함께 보여줍니다.

## 2) 지금 바로 확인할 화면 변화

1. 뉴스 탭에서 `Only 코인속보` 대신 `내 자산 뉴스` 탭이 보입니다.
2. 뉴스 카드 하단 문구가 `내 자산 영향지수 xx/100` 형태로 보입니다.
3. 실시간 탭은 최근 뉴스 중심으로 더 깔끔해지고, 같은 이슈 반복 노출이 줄어듭니다.

## 3) 남은 1개 작업 (중요)

`market_news` 품질 컬럼 마이그레이션을 원격 DB에 반영해야 합니다.

- 파일: `supabase/migrations/20260222113000_market_news_quality_signals.sql`
- 현재 `supabase db push`는 기존 레거시 마이그레이션 순서 문제로 자동 반영이 막혀 있습니다.

### 가장 쉬운 적용 방법

1. Supabase Dashboard 접속
2. SQL Editor 열기
3. 위 파일 전체 내용을 복사/붙여넣기
4. Run 실행

적용되면 품질 점수 컬럼(`news_quality_score` 등)이 DB에 생성되어, 뉴스 품질 랭킹 기능이 완성됩니다.

## 4) 운영 체크 (매일 1분)

1. 뉴스 탭 최신 기사 시간이 2~3시간 이상 밀리면 `daily-briefing Task J` 재실행
2. 같은 제목의 반복 뉴스가 많아지면 피드 소스 우선순위 재조정
3. `내 자산 뉴스` 탭이 비어 있으면 보유 자산 티커 입력 상태 점검
